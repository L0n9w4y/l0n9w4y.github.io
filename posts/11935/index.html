0x01 Linux数据项

apparmor_events （事件表）

跟踪 AppArmor 事件

apparmor_profiles

跟踪活动的 AppArmor 配置文件

apt_sources

APT 存储库或软件频道的当前列表

arp_cache

地址解析缓存，静态和动态（来自 ARP、NDP）

atom_packages

列出目录中的所有原子包或系统中全局安装的所有原子包

authorized_keys

以行分隔的 authorized_keys 表

block_devices

块（缓冲访问）设备文件节点：磁盘、ramdisk 和 DMG 容器

bpf_process_events （事件表）

跟踪时间/动作流程执行

bpf_socket_events （事件表）

跟踪网络套接字打开和关闭

cpu_time

显示来自 /proc/stat 文件的有关 cpu 内核在系统不同部分花费的时间的信息

cpuid

cpuid ASM 调用中的有用 CPU 功能

crontab

来自系统和用户 crontab 的行解析值

curl

执行一个 http 请求并返回有关它的统计信息

deb_packages

已安装的 DEB 包数据库

device_file

类似于文件表，但使用 TSK 并允许块地址访问

device_hash

类似于哈希表，但使用 TSK 并允许块地址访问

device_partitions

使用 TSK 枚举有关磁盘设备上的分区的详细信息

dns_resolvers

此主机使用的解析器

docker_container_envs

Docker 容器环境变量

docker_container_fs_changes

更改容器文件系统上的文件或目录

docker_container_mounts

Docker 容器挂载

docker_container_networks

Docker 容器网络

docker_container_ports

Docker 容器端口

docker_container_processes

Docker 容器进程

docker_containers

Docker 容器信息

docker_image_history

Docker 镜像历史信息

docker_images

Docker 镜像信息

docker_info

Docker 系统信息

docker_networks

Docker 网络信息

docker_version

Docker 版本信息

docker_volumes

Docker 卷信息

elf_dynamic

ELF 动态部分信息

elf_info

ELF 文件信息

elf_sections

ELF 部分信息

elf_segments

ELF 段信息

elf_symbols

ELF 符号列表

etc_hosts

行解析的 /etc/hosts

extended_attributes

返回文件的扩展属性（类似于 Windows ADS）

file

交互式文件系统属性和元数据

file_events

跟踪配置数据中指定的文件的时间/操作更改

groups

本地系统组

hardware_events（事件表）

来自 UDEV 或 IOKit 的硬件 (PCI/USB/HID) 事件

hash

文件系统哈希数据

iptables

Linux IP 包过滤和 NAT 工具信息

kernel_info

基本的活动内核信息

kernel_modules

已加载并位于加载搜索路径中的Linux 内核模块

known_hosts

以行分隔的 known_hosts 表

last

系统登录和注销记录

listening_ports

具有监听（绑定）网络套接字/端口的进程

logged_in_users

在系统上具有活动 shell 的用户

lxd_certificates

LXD 证书信息

lxd_cluster

LXD 集群信息

lxd_cluster_members

LXD 集群成员信息

lxd_images

LXD 镜像信息

lxd_instance_config

LXD 实例配置信息

lxd_instance_devices

LXD 实例设备信息

lxd_instances

LXD 实例信息

lxd_networks

LXD 网络信息

lxd_storage_pools

LXD 存储池信息

magic

幻数识别库表

memory_array_mapped_addresses

与物理内存数组的地址映射相关的数据

memory_arrays

与操作以形成存储器地址的存储器设备的集合相关联的数据

memory_device_mapped_addresses

与物理内存设备的地址映射相关的数据

memory_info

以字节为单位的内存信息

memory_map

操作系统内存区域映射

mounts

系统安装的设备和文件系统（不是特定于进程的）

npm_packages

列出目录中或全局安装在系统中的所有 npm 包

os_version

包含操作系统名称和版本的单行

process_envs

每个进程的环境变量的键/值表

process_events （事件表）

跟踪时间/动作流程执行

process_file_events （事件表）

使用审计服务的文件完整性监视器实现

process_memory_map

进程内存映射文件和伪设备/区域

process_namespaces

在主机系统上运行的进程的 Linux 命名空间

process_open_files

每个进程的文件描述符

process_open_pipes

每个进程的管道和伙伴进程

process_open_sockets

在系统上打开网络套接字的进程

processes

主机上所有运行的进程信息

python_packages

安装在系统中的 Python 包

routes

主机系统的活动路由表

rpm_package_files

当前安装在主机系统上的 RPM 软件包

rpm_packages

当前安装在主机系统上的 RPM 软件包

seccomp_events （事件表）

跟踪 seccomp 事件的虚拟表

selinux_events （事件表）

跟踪 SELinux 事件

selinux_settings

跟踪活动的 SELinux 设置

shadow

本地系统用户加密密码和相关信息。请注意，您通常需要超级用户权限才能访问 /etc/shadow

shared_memory

操作系统共享内存区域

shell_history

每个用户 .*_history 数据的行分隔（命令）表

socket_events （事件表）

跟踪网络套接字打开和关闭

ssh_configs

已解析的 ssh_configs 表

startup_items

应用程序和二进制文件设置为用户/登录启动项

sudoers

通过 sudo 以其他用户身份运行命令的规则

suid_bin

suid 二进制文件在公共位置

system_controls

sysctl 名称、值和设置信息

system_info

用于识别的系统信息

uptime

自上次启动以来经过的跟踪时间。一些系统将此作为日历时间，一些作为运行时间

user_events （事件表）

从审计框架跟踪用户事件

user_groups

本地系统用户组关系

user_ssh_keys

返回用户 ~/.ssh 目录中的私钥以及它们是否被加密

users

本地用户帐户

yara_events

跟踪文件或 PID 的 YARA 匹配

cloud_instance_metadata

Cloud 实例元数据

yum_sources

Yum 存储库或软件频道的当前列表
